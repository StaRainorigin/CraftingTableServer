# 开启一个模组服务器

## 简介
模组服务器允许你添加各种模组(Mod)来丰富游戏内容 常见的模组服务端加载器有 Fabric, Forge 和 NeoForge 

## 下载服务端核心 (安装器)

与原版或插件服不同，模组服通常需要下载一个**安装器(Installer)**来自动下载库文件并构建服务端核心 

- **Fabric**: [Download Fabric](https://fabricmc.net/use/server/)
- **Forge**: [Minecraft Forge Downloads](https://files.minecraftforge.net/)
- **NeoForge**: [NeoForged Downloads](https://neoforged.net/)

## 安装服务端

### Windows / Linux (带图形界面)
1. 双击下载好的安装器 (`.jar` 或 `.exe` 文件) 
2. 选择 **"Install Server"** (安装服务端) 
3. 点击 "..." 选择你要安装到的空文件夹 
4. 点击确定开始安装 
5. 等待安装完成 (出现 "Successfully installed..." 字样) 

### 命令行 (无图形界面)
在安装器目录下打开终端/CMD:

```shell
# Fabric
java -jar <fabric-installer>.jar server -mcversion <游戏版本> -dir <安装目录>

# Forge / NeoForge
java -jar <installer>.jar --installServer
```

## 启动服务器

安装完成后，你会发现目录下多了很多文件 启动方式略有不同 

### Fabric
通常会生成一个 `fabric-server-launch.jar` 
参考 [快速开始](快速开始.md) 中的启动脚本，将 `<服务端核心名>.jar` 替换为 `fabric-server-launch.jar` 

### Forge / NeoForge (1.17+)
高版本的 Forge/NeoForge 安装后通常会直接生成 `run.bat` (Windows) 和 `run.sh` (Linux) 
- **Windows**: 直接双击 `run.bat` 即可启动 
- **Linux**: 运行 `./run.sh` 

> **注意**: 如果你需要修改内存大小，请编辑生成的 `user_jvm_args.txt` 文件，修改 `-Xmx` 和 `-Xms` 参数 

### Forge (1.16.5 及以下)
你会得到一个 `forge-<版本>-universal.jar` 
参考 [快速开始](快速开始.md) 中的启动脚本，将 `<服务端核心名>.jar` 替换为这个 universal jar 文件的名字 

## 同意 EULA
首次启动后，服务器会停止并生成 `eula.txt` 
打开该文件，将 `eula=false` 改为 `eula=true`，然后再次启动服务器
